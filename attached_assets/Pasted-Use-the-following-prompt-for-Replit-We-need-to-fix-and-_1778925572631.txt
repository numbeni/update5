3. Remove SSL Checks From Main Site Monitoring

SSL checks must no longer run as part of the main website monitoring sweep.

The main monitoring system should not perform SSL tests while checking websites.

Reason:
SSL is now managed separately inside the SSL Certificates section.

From now on:

main site monitoring must not run SSL checks

4. SSL Check Scheduling Rules

SSL Check On All Sites is a heavier operation and must not interfere with the main monitoring engine.

Implement correct scheduling and locking behavior.

Rules:

automatic SSL scan should run once every 24 hours
automatic SSL scan should only run when the main monitoring system is idle or in cooldown
automatic SSL scan must not start while a monitoring sweep is actively running
if the main monitoring sweep is running, delay the SSL scan until a safe idle/cooldown moment
once SSL Check On All Sites starts, the main monitoring sweep must not start again until the SSL scan is finished
main monitoring should wait until the SSL scan completes
after SSL scan finishes, main monitoring may resume normally
SSL checks should run gradually with spacing/delay between sites
avoid heavy parallel SSL checks

This rule applies to:

automatic 24-hour SSL scan
manual Run SSL Check On All Sites

Important exception:

Manual SSL Check for one single site/domain can run independently and does not need to block monitoring.

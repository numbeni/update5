- Add configurable thresholds:
  - failureThresholdBeforeIncident
  - recoveryThresholdBeforeResolve
- Recommended defaults:
  - failureThresholdBeforeIncident = 2
  - recoveryThresholdBeforeResolve = 2
- Only create incident after repeated failures.
- Only auto-resolve incident after repeated successful checks.
- Store or calculate consecutive failure/success counts.
- Show in UI when an incident was not created yet:
  “Failure count 1/2”

- Make threshold configurable from Settings if possible.

Check 1 DOWN:
- Save check result.

- Create incident.

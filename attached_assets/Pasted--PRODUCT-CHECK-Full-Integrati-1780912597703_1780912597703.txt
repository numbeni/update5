## 1. Fix: unknown and failed both mean negative result

**Current behavior (WRONG):**
When product check cannot find any `/product/` links → returns `unknown`.
When it finds links but none respond → returns `failed`.
`unknown` currently has no status impact — site stays `up`.

**Required behavior:**
Both `unknown` AND `failed` must be treated as a negative product check result.
Both must trigger the same outcome:
- `result.overallStatus = "degraded"`
- `result.errorType = "product_page_issue"`
- `result.errorMessage = "Product pages not responding"`

In `engine.ts`, change the status impact condition from:
```ts
if (pcResult.status === "failed" && ...)
```
to:
```ts
if ((pcResult.status === "failed" || pcResult.status === "unknown") && ...)
```

Do not change `product-check.ts`. The `unknown` status remains valid as a return value. Only the engine's reaction to it changes.

## 2. Fix: Incidents must use product_page_issue type, not application_issue

**Current behavior (WRONG):**
In `incidents.ts`, when `overallStatus === "degraded"`, incident type is always forced to `"application_issue"` regardless of `errorType`.

**Required behavior:**
When `errorType === "product_page_issue"`, incident type must be `"product_page_issue"`.

Fix the `getIncidentType` function in `incidents.ts`:
```ts
if (check.overallStatus === "degraded") {
  if (check.errorType === "product_page_issue") return "product_page_issue";
  return "application_issue";
}
```

Incident title for `product_page_issue` must be: **"Product pages unreachable"**
Incident severity must be: `"warning"`

## 7. Fix: Reduce Live Console noise for product check

**Current behavior (WRONG):**
Product check emits 5 separate console events per site:
1. `product check: starting`
2. `product check: fetching homepage`
3. `product check: scanning for /product/* links`
4. `product check: checking /product/example/ (1/3)`
5. `product check: ok (1/1 ok) — 35ms`

This floods the console and makes it hard to read.

**Required behavior:**
Emit only **2** console events per product check:

**Event 1** — at start:
```
product check → <sitename>
```

**Event 2** — at completion (one of):
```
product ok — /product/example-slug/ (42ms)
product failed — no pages responded (312ms)
product unknown — no /product/* links found (701ms)
product error — homepage unreachable (timeout)
```

Remove all intermediate events (`fetching_homepage`, `discovering`, `probing` per URL).

Update `engine.ts` only. Do not change `product-check.ts`.

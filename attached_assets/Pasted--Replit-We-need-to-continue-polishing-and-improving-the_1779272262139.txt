FRONTEND CHANGES
================

1. Critical / Down Site Card Text Colors

In the Critical / Down Sites cards, update text colors:

* The text like:
  “X consecutive failures”
  or
  “fail number X”
  should be red, bold, and attention-grabbing, but not an eye-burning bright red.
  Use a professional dark-theme friendly red.

* The domain text color in these cards should match the server-section muted text color:
  hsl(215 16% 47%)

* The actual failure reason / down reason should keep its current warning-like color:
  oklch(75% 0.183 55.934)

Goal:
Operators should immediately notice repeated failures, but the card should still look professional and readable.

2. Make Card Widths Consistent Across Dashboard
===============================================

There is a major frontend issue where card widths become inconsistent in different states.

Examples:

* In server sections, when one site becomes unstable and an extra line appears, card widths become misaligned.
* In Critical / Down Sites, when a new card is added before its “X consecutive failures” line appears, the card width differs from others.
* In general, dashboard cards should remain visually aligned and equal-width.

Fix this globally.

Requirements:

* Site cards should keep consistent widths regardless of status.
* Cards should reserve space for status metadata lines even if the value is not available yet.
* Layout should not jump when new data arrives.
* Critical cards, server grid cards, up/slow/unstable/down cards should all remain aligned.
* Avoid layout shift during sweeps and realtime updates.

For UP cards, add a third informational line similar to slow/unstable/down cards, so all cards remain visually consistent.

Suggestion:
For UP cards, show something like:

* “No recent failures”
* or “Clean for X checks”
* or “X consecutive healthy checks”

Use a subtle color between white and green to indicate positive stability.

This helps operators understand which sites/servers are consistently healthy.

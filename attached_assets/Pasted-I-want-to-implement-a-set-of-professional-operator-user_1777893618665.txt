  - Away
  - Busy
  - In Work Shift

Operator One | Operator | ⚪ Offline — last seen 12 minutes ago

### Presence logic

Implement a reliable heartbeat mechanism.

- User can manually choose:
  - Online
  - Away
  - Busy
  - In Work Shift

### Auto logout logic

- Invalidate/revoke their session
- Next time the browser checks auth, redirect to login
- Do this safely server-side, not only in frontend

## 9. Add per-site Pause / Resume monitoring

Add the ability to pause monitoring for a specific site.

Important distinction:
This is different from global monitoring pause.

### Behavior

If a site is paused:
- It should be excluded from automatic monitoring sweeps
- It should not appear in automatic queue
- It should not create automatic incidents while paused
- It should not affect global monitoring
- Manual checks should still be allowed from the site detail page
- It should still be possible to manually add it to the queue
- If manually added to queue, the queue should process it as a manual check
- It remains excluded from automatic monitoring until resumed

### UI requirements

Add site pause/resume action in:
- Monitored Sites list right-click menu
- Monitored Sites grid right-click menu
- Bulk selection actions in List View
- Bulk selection actions in Grid View if multi-select exists
- Site detail page action button if appropriate

Show paused state clearly in both list and grid:
- Add “Paused” badge near site name
- Use a subtle glass/transparent overlay style
- Keep it readable in both light and dark mode
- In grid view, paused cards should visually feel inactive but still clickable
- In list view, paused rows should be visually distinct

Suggested style:
- translucent glass overlay
- muted text
- paused badge
- no aggressive red color, because paused is intentional

### Backend requirements

Add field if needed:
sites:
- monitoringPaused: boolean
- monitoringPausedAt
- monitoringPausedBy

Monitoring engine:
- Automatic sweeps must skip paused sites
- Automatic queue must not include paused sites
- Manual check endpoint should still allow checking paused site
- Manual add-to-queue should still allow paused site if explicitly user-triggered

Audit:
- Log site paused/resumed
- Include site ID and site name

Permissions:
- founder/admin/operator can pause/resume sites
- viewer cannot

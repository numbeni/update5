WHAT SHOULD APPEAR IN THE TERMINAL
==================================

The terminal should display realtime connectivity-related operations such as:

* ping execution
* DNS resolve attempts
* target switching
* retries
* RTT measurements
* RTT degradation warnings
* ONLINE / DEGRADED / OFFLINE transitions
* sweep pause/resume due to connectivity state
* connectivity recovery
* retry countdowns
* diagnostic checks
* backend decision flow

Example style of logs:

[13:04:30] ping soft98.ir
Reply from 172.x.x.x: time=45ms

[13:04:31] Primary connectivity target ONLINE

[13:04:35] ping [www.varzesh3.com](http://www.varzesh3.com)
Request timeout

[13:04:36] Trying secondary target...

[13:04:40] DNS resolve google.com
Resolved: 142.x.x.x

[13:04:42] Connectivity state changed:
ONLINE → DEGRADED

[13:04:45] Retrying in 5s...

==================================================
OFFLINE / DEGRADED MODE
=======================

When connectivity becomes OFFLINE or DEGRADED:

* the terminal must stay active
* retry loops should appear live
* monitoring pause events should appear
* checkpoint preservation should appear
* recovery/resume events should appear

Example:

All primary targets failed
Connectivity status: OFFLINE

Monitoring paused
Sweep checkpoint preserved

Retrying connectivity in 5s...

Connectivity restored
Resuming monitoring sweep...

==================================================
RTT TRENDING INTEGRATION
========================

If RTT trending exists, terminal logs should also show latency degradation insights.

Examples:

Average RTT increased from 52ms → 148ms

RTT Trend: rising

The terminal should help operators understand not only whether connectivity exists, but also whether network quality is degrading before full failure occurs.

7. Improve Critical / Down Banner Logic
=======================================

Finalize the new Down/Critical Banner behavior.

Current intended architecture:

* first server pass
* second pass for down sites inside same server
* final global down recheck

Behavior rules:

If a site becomes DOWN after first+second confirmation:

* move it to Critical Banner
* remove it from server accordion section
* avoid duplicate rendering

If during Final Down Recheck the site recovers:

* immediately remove it from Critical Banner
* move it back into its original server section
* update server metrics live
* synchronize all views

If still DOWN:

* keep it inside Critical Banner only

Important:
A DOWN site should only exist in ONE visual location at a time.

No duplicate rendering between:

* Critical Banner
* Grid
* Compact
* List
* Server accordion

All views must consume one shared normalized monitoring state.

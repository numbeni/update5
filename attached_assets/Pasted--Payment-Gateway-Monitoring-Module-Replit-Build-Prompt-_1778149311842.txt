# Payment Gateway Monitoring Module — Replit Build Prompt

You are expanding an existing NOC / Hosting Monitoring platform.

The current system already contains:

* Website monitoring
* DNS checks
* HTTP/TCP/SSL monitoring
* Incident management
* Diagnostics pages
* Alert system
* Dashboard
* PostgreSQL + Drizzle ORM
* Frontend operator panel
* Backend monitoring engine

Your task is to design and implement a NEW module called:

# Payment Gateway Monitoring

The purpose of this module is:

* Monitor the public availability and health of Iranian payment gateways
* Allow operators to manually or bulk-import payment gateways
* Link payment gateways to customer websites
* Show gateway health inside the related website diagnostics section
* Help operators understand which customer websites are affected when a gateway becomes degraded or unavailable

IMPORTANT:
This is NOT merchant credential monitoring.
DO NOT request bank account data, payment credentials, merchant IDs, API secrets, or customer payment access.

This module is Phase 1 only:
Public gateway health monitoring.

---

# Core Goals

The system should monitor public payment gateway infrastructure such as:

* ZarinPal
* Zibal
* NextPay
* IDPay
* Vandar
* Mellat
* SEP / Saman
* Sadad
* Parsian
* DigiPay
* Asan Pardakht
* Other Iranian PSPs

The system should determine:

* Is the gateway reachable?
* Is DNS healthy?
* Is SSL healthy?
* Is the main payment page reachable?
* Is API latency abnormal?
* Is the gateway degraded?
* Is the gateway potentially causing customer website payment failures?

# 2. Manual Gateway Creation

Operators must be able to manually add gateways.

Fields:

* Gateway name
* Provider type
* Main domain
* API endpoint URL

* Tags
* Notes

# 3. Bulk Import

Operators MUST be able to bulk import gateways.

Supported methods:

* Paste domains list
* CSV import
* JSON import
* Multi-line textarea import

Example:

zarinpal.com
gateway.zibal.ir
sep.shaparak.ir

System should automatically:

* detect DNS hostname
* normalize URLs
* classify gateway type if possible

---

# 4. Gateway Health Checks

Implement automated checks:

## DNS Check

* resolve gateway domain
* detect timeout/failure

## SSL Check

* certificate validity
* expiration
* issuer
* chain issues

## HTTP/API Reachability

* response status
* latency
* timeout detection

## Payment Page Reachability

* ensure payment page loads
* detect major outages

## Gateway Status Logic

Possible statuses:

* UP
* DEGRADED
* PARTIAL_OUTAGE
* DOWN
* UNKNOWN

# 6. Website Diagnostics Integration

Inside each monitored website diagnostics page,
add a new section:

Payment Gateway Diagnostics

Example:

Payment Gateways:

* ZarinPal → UP
* SEP → DEGRADED

* Is the customer website issue caused by the gateway?

This is one of the most important goals of the module.

- [Server color gradient support](server-color-gradient.md) — server.color can be a CSS gradient string; use `background:` not `backgroundColor:` everywhere to support both.
- [Site rename endpoint](site-rename.md) — PATCH /api/sites/:id/rename updates name, url, host; operator+ required; host auto-extracted via `new URL(url).hostname`.
- [Dashboard accordion state](dashboard-accordion.md) — openServers is `Set<number>` (not `"all"`); empty set = all collapsed (default); sortedServers drives allOpen + toggleAllServers.
- [All-in-one import/export](all-in-one-io.md) — POST /api/sites/all-in-one-import + GET /api/sites/all-in-one-export; format: "Name | CODE\nurl\nurl\n"; matches server by code first.
- [Shop fallback architecture](shop-fallback.md) — /shop logic centralised in applyShopOverride() called INSIDE runAndStoreCheck; result mutated before DB insert so incident/streak/alert logic all see correct status. alsoShop DB flag enables it on every check; checkShopFallback option enables it for Phase 2 & Phase 3 sweeps. Old separated shopFallbackOk() removed.
- [Product check monitoring integration](product-check-integration.md) — runProductCheck called inside runAndStoreCheck after shop override; only when site.productCheckEnabled AND status is UP/SLOW/DEGRADED; failed product check → overallStatus="degraded", errorType="product_page_issue"; emits "product" console events; ConsoleEventType "product" added to all 3 type locations (backend + 2 generated files).
- [Product check toggle endpoint](product-check-toggle.md) — dedicated PATCH /api/sites/:id/product-check (requires operator+, body: {enabled: boolean}); frontend uses this, NOT the generic PATCH /sites/:id; generic PATCH had silent failures when disabling.
- [Critical bar vs Product Issues bar](critical-bar-logic.md) — criticalSites excludes degraded+product_page_issue; productIssueSites (errorType=product_page_issue) go to ProductIssuesBanner only; both sets merged into criticalSiteIds to hide from server accordion; CompactSiteCard Product badge always visible when productCheckEnabled=true.
- [onRefetch force-refetch pattern](on-refetch-pattern.md) — staleTime:20000 on useListSites means invalidateQueries alone won't re-fetch; onRefetch must also call refetchQueries to guarantee an immediate network request after any toggle.
- [AppSettings index signature](appsettings-index.md) — both AppSettings and UpdateAppSettingsBody generated types need `[key: string]: unknown` index signature so custom settings (alertSweepDownSites, ncAlertSweepDownSites, etc.) pass TypeScript without casting everywhere.
- [Sweep-end notification architecture](sweep-end-notif.md) — alertSweepDownSites = server-side toggle controlling SSE broadcast; ncAlertSweepDownSites = Nextcloud Talk per-site; rootCause "sweep_end_down" avoids cooldown clash with regular site_down alerts; alertSweepDownSites toggle lives in BrowserNotificationsSection (not Alerts/NC section).
